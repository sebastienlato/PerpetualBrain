# Prompt Library

## New App Kickoff
Build a local-first, polished app for the described product. Start by reading the repo, identify the existing stack, implement the smallest coherent version, run build checks, and report files changed, verification, limitations, and next steps.

## New Game Kickoff
Build a browser game prototype with a proven game framework. Separate simulation from UI, include review routes for visual states, verify with screenshots, and keep the first playable loop small but complete.

## Phase Implementation
Implement this phase only: [PHASE]. Keep changes scoped to the phase. Run build/typecheck, verify any visual surface with screenshots, and report remaining risks.

## Bug Fix
Fix this bug only: [BUG]. Do not add unrelated features. Identify the root cause, patch the smallest safe area, run the most relevant check, and describe the verification.

## UI Polish Pass
Improve the visible UI for [SURFACE]. Keep the existing product model intact. Focus on spacing, hierarchy, responsive behavior, copy clarity, empty states, and screenshot verification.

## Asset Generation Pass
Generate or prepare assets for [FEATURE]. Do not wire generated assets into production until they are reviewed in-context. Include naming, dimensions, anchor expectations, and approval screenshots.

## Refactor Pass
Refactor [AREA] to reduce concrete complexity while preserving behavior. Avoid broad rewrites. Add or update tests where behavior could regress.

## QA Pass
Perform a QA pass on [AREA]. Run available checks, inspect responsive layouts, capture important screenshots, and return prioritized findings with file references.

## Release Prep
Prepare [PROJECT] for release. Validate build, docs, basic accessibility, known limitations, version notes, and rollback risks.

## Git Commit Summary
Summarize the current changes into a concise commit message with a subject line and bullet body. Mention tests run and any known limitations.
