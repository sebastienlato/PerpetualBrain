# Codex Workflow

## Kickoff
Give Codex the project context, current goal, exact files or routes involved, constraints, and acceptance criteria. Include recent decisions and lessons when they are relevant.

## During Work
- Ask for implementation, not just a plan, unless the phase is exploratory.
- Keep phase boundaries explicit.
- Require build and visual verification for UI work.
- Ask for known limitations and next recommended phase.

## Handoff
Every handoff should include:
- Files changed
- Commands run
- Screenshots or QA notes when relevant
- Remaining risks
- Next recommended task
