# Design Standards

## Product Feel
Build quiet, dense, polished devtools. Avoid marketing-page composition inside tools. Prioritize scanning, comparison, repeated action, and fast copy/paste workflows.

## Layout
- Use a stable sidebar and predictable page headers.
- Keep cards for repeated items, summaries, and framed tools.
- Avoid nested cards.
- Make primary actions visible without overwhelming the workspace.

## Visual Language
- Dark mode first.
- Use restrained gradients and subtle borders.
- Use icons for common commands.
- Keep text compact and specific.
- Do not expose roadmap language in user-facing screens unless the screen is explicitly about planning.
